# リファクタリング・クリーン

テスト検証によるデッドコードの安全な特定と削除:

1. デッドコード分析ツールを実行:
   - knip: 未使用のエクスポートとファイルを検出
   - depcheck: 未使用の依存関係を検出
   - ts-prune: 未使用のTypeScriptエクスポートを検出

2. 包括的なレポートを .reports/dead-code-analysis.md に生成

3. 検出結果を重要度別に分類:
   - 安全: テストファイル、未使用ユーティリティ
   - 注意: APIルート、コンポーネント
   - 危険: 設定ファイル、メインエントリーポイント

4. 安全な削除のみを提案

5. 各削除の前に:
   - 全テストスイートを実行
   - テストがパスすることを確認
   - 変更を適用
   - テストを再実行
   - テストが失敗した場合はロールバック

6. クリーンアップ項目のサマリーを表示

テストを実行せずにコードを削除してはならない！
